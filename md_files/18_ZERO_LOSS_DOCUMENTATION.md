# 📝 Zero-Loss Documentation Rule

## Objective
To ensure that NO information, decision, or logic is ever lost. Every chat interaction that results in a change or a decision must be documented.

## AI Instructions
1. **Real-Time Logging**: Every code change MUST be logged in `02_project_status.md` immediately with the date, time (IST), and reason.
2. **Decision Tracking**: If a design choice is made (e.g., "Use black background for buttons"), record it in `07_styling.md` or `04_architecture.md`.
3. **TODO Updates**: Every new requirement or idea mentioned by the USER must be added to `03_TODO.md` as a task.
4. **No Assumptions**: If you are unsure about a detail, do not guess. Ask the USER and document the answer.

---
*Status: Active. This rule ensures the project's state is always 100% transparent.*
